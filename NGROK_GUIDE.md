# 🌐 Инструкция по запуску через ngrok

Использование **ngrok** позволяет получить публичный адрес для твоего сайта. На бесплатном тарифе ngrok обычно позволяет держать только один активный туннель, поэтому тебе может понадобиться два разных аккаунта или комбинирование с localtunnel.

### ⚠️ Обязательный шаг: Виртуальное окружение
Перед запуском бэкенда **всегда** активируй виртуальное окружение, иначе зависимости не будут найдены.

В терминале бэкенда:
```powershell
cd f:\TmWork\backend
# Активация venv
.\.venv\Scripts\activate
```

---

### 👣 Пошаговый запуск

#### 1. Запуск Бэкенда
1. Открой терминал и активируй `.venv/` (как показано выше).
2. Запусти сервер:
   ```powershell
   python manage.py runserver
   ```

#### 2. Публикация Бэкенда через ngrok
В новом терминале запусти:
```powershell
ngrok http 8000
```
> **СКОПИРУЙ** ссылку (например, `https://a1b2-c3d4.ngrok-free.app`)

#### 3. Настройка Фронтенда
Открой файл `f:\TmWork\frontend\.env` и вставь ссылку:
```env
VITE_API_URL=https://a1b2-c3d4.ngrok-free.app
```

#### 4. Запуск Фронтенда
В новом терминале:
```powershell
cd f:\TmWork\frontend
npm run dev
```

#### 5. Публикация Сайта через ngrok
Если у тебя есть возможность запустить второй туннель:
```powershell
ngrok http 3000
```
> Эту финальную ссылку отправляй другу.

---

### 💡 Советы
1. **Проверка IP**: При первом переходе по ссылке ngrok может показать страницу-заглушку. Нужно нажать кнопку **"Visit Site"**.
2. **Фолбек**: Если ты не используешь ngrok, просто оставь `VITE_API_URL` в `.env` пустым — проект автоматически переключится на `127.0.0.1:8000`.
3. **Ошибки**: Если бэкенд ругается на "Disallowed Host", я уже исправил это в `settings/base.py`, установив `ALLOWED_HOSTS = ['*']`.
