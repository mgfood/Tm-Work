# 🧪 Руководство по тестированию TmWork

В этом документе собраны все команды для запуска тестов в проекте.

---

## 1. Фронтенд (React + Vitest + Playwright)

Все команды запускаются из папки `frontend/`.

### Модульные тесты (Unit Tests — Vitest)
*Проверяют отдельные компоненты и логику в изоляции.*

*   **В терминале (быстро):**
    ```bash
    npm run test
    ```
*   **Графический интерфейс (удобно для отладки):**
    ```bash
    npm run test:ui
    ```

### Сквозные тесты (E2E Tests — Playwright)
*Проверяют работу всего сайта целиком (как робот-пользователь).*

*   **В терминале (фоновый режим):**
    ```bash
    npm run test:e2e
    ```
*   **Интерактивный UI (лучший способ, с историей действий):**
    ```bash
    npx playwright test --ui
    ```
*   **Увидеть «робота» в живую (откроется обычное окно браузера):**
    ```bash
    npx playwright test --headed
    ```
*   **Открыть отчет последнего теста (если что-то упало):**
    ```bash
    npx playwright show-report
    ```

---

## 2. Бэкенд (Django + Pytest)

Команды запускаются из папки `backend/` с активированным виртуальным окружением.

*   **Все тесты бэкенда:**
    ```bash
    pytest
    ```
*   **С подробным выводом:**
    ```bash
    pytest -v
    ```
*   **С проверкой покрытия кода (Coverage):**
    ```bash
    pytest --cov=.
    ```

---

## 💡 Советы
1.  **Для E2E тестов** обязательно должны быть запущены оба сервера (`npm run dev` и `runserver`).
2.  Если тест падает, используйте **Playwright UI**, чтобы кликнуть на ошибку и увидеть скриншот системы в этот момент.
3.  Используйте `npx playwright install`, если браузеры еще не установлены в вашей системе.
