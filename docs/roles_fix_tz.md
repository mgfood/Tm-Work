# ТЗ: Исправление логики ролей пользователей

## 1. Выявленные проблемы
1. **Ошибка UI (Highlighting)**: Активная роль не подсвечивается в профиле. Причина: во фронтенде идет проверка `user.roles.includes('ROLE_NAME')`, но массив ролей содержит объекты `{id, name}`, а не строки.
2. **Ошибка Admin Panel**: Невозможно изменить роли через админку. Причина: некорректная логика определения "наличия роли" в `UserEditModal.jsx`, что приводит к конфликтам в API (попытка добавить уже существующую роль).
3. **UX**: Отсутствует визуальное различие между кнопками ролей, когда одна из них активна.

## 2. Предлагаемые изменения

### 2.1. Frontend: Глобальные правки в профиле
- В `ProfileSidebar.jsx` и подобных компонентах изменить логику проверки:
  ```javascript
  // Было
  user.roles.includes('CLIENT')
  // Стало
  user.roles.some(r => r.name === 'CLIENT')
  ```
- Добавить визуальное выделение активной роли: более яркий бордер, фоновый градиент или галочка (Check icon).

### 2.2. Frontend: Admin Panel (`UserEditModal.jsx`)
- Исправить инициализацию стейта ролей: мапить массив объектов ролей в булевы флаги.
- Исправить логику `handleSubmit`: корректно сравнивать текущие (в стейте) и оригинальные (в объекте user) роли для вызова `assignRole` / `removeRole`.

### 2.3. Backend: Валидация
- Убедиться, что `toggle-role` в `UserViewSet` корректно обрабатывает запросы и обновляет ManyToMany связь.
- Добавить логирование изменений ролей в `AdminLog`.

## 3. Требования к UI
- Активная роль должна иметь статус `primary` (синий/фиолетовый цвет).
- Неактивная роль должна быть приглушенной.
- При нажатии на роль должно выводиться подтверждение, если это ведет к смене контекста работы.
