# TmWork Frontend Documentation

Эта документация описывает архитектуру и стандарты разработки фронтенд-части проекта TmWork.

## 🏗 Архитектура

Проект построен на **React 18** с использованием сборщика **Vite**.

### Структура папок
- `src/api/` — Сервисы для работы с бэкендом (Axios клиенты, методы API).
- `src/components/` — Переиспользуемые UI-компоненты (Кнопки, Инпуты, Шапка).
- `src/context/` — Глобальное состояние (Аутентификация, темы).
- `src/pages/` — Страницы приложения, сгруппированные по функционалу.
- `src/index.css` — Глобальные стили и токены Tailwind.

## 🎨 Дизайн-система

Мы используем **Tailwind CSS** для стилизации. Основные принципы:
1. **Премиальность**: Использование теней `shadow-premium`, скруглений `rounded-2xl` и "стеклянных" эффектов `glass-morphism`.
2. **Адаптивность**: Mobile-first подход. Все страницы должны корректно отображаться на мобильных телефонах.
3. **Цвета**: 
   - `primary-600` (#2563eb) — основной акцентный цвет.
   - `slate-900` — текст и темные элементы.
   - `slate-50` — фон приложения.

## 🔐 Аутентификация

Реализована через `AuthContext` и JWT токены.
- Токены хранятся в `localStorage`.
- Axios-интерцептор (`src/api/client.js`) автоматически добавляет `Authorization` заголовок и обновляет токены при истечении (Refresh token rotation).

## 📡 API Интеграция

Для каждого Django-приложения создан соответствующий сервис:
- `authService.js` — Вход, регистрация, выход.
- `jobsService.js` — Создание и поиск заказов.
- `proposalsService.js` — Отклики.
- `profilesService.js` — Профили и навыки.

### Пример использования:
```javascript
import jobsService from '../api/jobsService';

const fetchJobs = async () => {
  const data = await jobsService.getJobs();
  setJobs(data);
};
```

## 🛠 Команды разработки

- `npm run dev` — запуск локального сервера (порт 3000).
- `npm run build` — сборка проекта для продакшена.
- `npm run preview` — проверка собранного билда.

## 💡 Советы по разработке
- Используйте библиотеку `lucide-react` для иконок.
- Всегда проверяйте интерфейс в режиме эмуляции мобильного устройства в Chrome DevTools.
- Не дублируйте логику — выносите сложные расчеты в хуки или вспомогательные функции.
