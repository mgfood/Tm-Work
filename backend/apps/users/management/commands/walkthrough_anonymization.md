# Архитектура Мягкого Удаления и Анонимизации (Освобождение Email и защита данных)

## Что было сделано:

1.  **Моментальное освобождение Email**:
    *   Обновлен метод `delete_account` в `apps/profiles/views.py`.
    *   При удалении аккаунта почта пользователя моментально меняется на `deleted_<id>_<old_email>`.
    *   **Результат**: Пользователь сразу же может зарегистрировать новый аккаунт со своей старой почтой. Ошибки "Email уже занят" больше не будет.

2.  **Политика хранения данных (`ACCOUNT_RETENTION_DAYS`)**:
    *   В общие настройки Django (`config/settings/base.py`) добавлена переменная `ACCOUNT_RETENTION_DAYS`.
    *   По умолчанию она равна 30 дням. Администратор может изменить её в любой момент через переменные окружения (`.env`).

3.  **Скрипт глубокой Анонимизации (Удаление личных данных)**:
    *   Написана новая команда управления для сервера: `cleanup_deleted_accounts` (`apps/users/management/commands/cleanup_deleted_accounts.py`).
    *   Она решает проблему ограничений FinTech-приложений: физическое удаление (`DELETE FROM users`) невозможно из-за привязки транзакций платформы (срабатывает защита `on_delete=models.PROTECT`).
    *   Команда ищет все удаленные аккаунты старше 30 дней и:
        - Полностью очищает модель `User` (имя, фамилия).
        - Полностью очищает модель `Profile` (телефон, биография, локация, навыки, соц. сети и т.д.).
        - Навсегда удаляет все файлы **аватара** и работы **портфолио** с диска сервера.
        - Стирает всю переписку в чатах (удаляет сообщения, отправленные данным пользователем, вместе с вложениями).
    *   **Финансовая целостность сохранена**: "пустая оболочка" (развоплощенный анонимный пользователь без личных данных) остается привязанной к транзакциям, так что администратор всегда может видеть исторический путь денег.

4.  **Безопасность и проверка**:
    *   Написан автоматический тест `test_cleanup_deleted_accounts.py`, который запускался и проверял, что скрипт корректно зачищает персональные данные только тех, кому пора (игнорируя недавно удаленных или активных). Все тесты пройдены успешно.

## Как использовать администратору:

Для автоматического запуска очистки достаточно заблокировать выполнение команды в **cron** (планировщик операционной системы сервера), например, на 3 часа ночи:
```bash
0 3 * * * cd /home/kali/Projects/Tm-Work/backend && uv run python manage.py cleanup_deleted_accounts >> /var/log/tmwork_cleanup.log 2>&1
```
Или вы можете запускать её вручную, когда посчитаете нужным.
