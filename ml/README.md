# 🧠 TmWork ML Engine

Эта папка содержит Machine Learning компоненты платформы TmWork. Модуль отвечает за интеллектуальный анализ данных, классификацию заказов и систему рекомендаций.

---

## 🛠 Технологический стек

* **Runtime:** Python 3.11+
* **Package Manager:** [uv](https://github.com/astral-sh/uv) (быстрая замена pip)
* **Libraries:** Scikit-learn, Pandas, Joblib (базовый набор)
* **Format:** Чистые `.py` скрипты (без Jupyter Notebooks для production-ready кода)

---

## 📁 Структура модуля

```text
ml/
├── data/               # Исходные данные для обучения (в .gitignore)
├── models/             # Скомпилированные веса моделей (.pkl, .pth)
├── src/                # Исходный код
│   ├── train.py        # Скрипт для обучения моделей
│   ├── inference.py    # КЛАСС ДЛЯ ИНТЕГРАЦИИ (используется бэкендом)
│   └── utils.py        # Препроцессинг текста и утилиты
├── tests/              # Тесты точности моделей
└── pyproject.toml      # Зависимости проекта (управляется через uv)

🚀 Быстрый старт для разработчика
1. Установка окружения
Мы используем uv для мгновенной синхронизации зависимостей. Если у тебя нет uv, установи его через pip install uv.

PowerShell
cd ml
uv sync
Это создаст папку .venv и установит все библиотеки из pyproject.toml.

2. Запуск предсказаний (Inference)
Для проверки работы модели без запуска всего сайта:

PowerShell
uv run python src/inference.py
🔌 Интеграция с Django (Backend)
Чтобы подключить ML-логику к серверу, используй класс predictor из src/inference.py.

Пример вызова в Django Service Layer:

Python
import sys
import os

# Добавление пути к ML модулю
sys.path.append(os.path.join(os.getcwd(), 'ml'))

from src.inference import model

# Пример получения категории
category = model.predict_category("Нужно разработать дизайн логотипа")
print(category) # Выведет: Design
📈 Планируемые фичи
Авто-категоризация: Определение категории заказа по описанию.

Smart Match: Подбор наиболее подходящих исполнителей для заказа.

Spam Filter: Модерация подозрительных объявлений.

⚠️ Правила разработки
Не пушить данные: Все файлы в data/ игнорируются гитом.

Веса моделей: Если модель весит >50MB, не добавляй её в репозиторий (используй внешнее хранилище).

Обновление зависимостей: После добавления новой библиотеки через uv add, обязательно сделай коммит файла uv.lock.
